HTML

  • The head elements contains information about the webpage, metadata, that helpes the browser to render the page correctly.
  • The body element represents the visible contennt shown to the user.
  • Content is contained between and opening tag and a closing tag
  • Elements nested inside others are child elements, and those that hold the child elements are the parent elements.
  • The < h1 >, < h2 >, < h3 >, < h4 >, < h5 >, and < h6 > elements represent the level of heading a given text block represents.
  • The < ul >, < ol >, and < li > elements represent unordered lists, ordered lists, and list items. In essence, these HTML elements represent bulleted or numbered lists.
  • The < a > element (which stands for anchor) creates links to the same webpage or other webpages. The href attribute points to the URL for the link
  • The < src > attribute defines the location, or source of the image file.
  • The < alt > attribute contains a text string that describes the appearance and functionality of the image.
  • HTML uses the id attribute as a way to identify elements and select them.

Git

  • Every Git repository starts with a main branch, and feature branches are when a new branch is created to add something to our project.
  • Prefix Git commands with 'git'
  • 'git status': checks what branch we are currently on
  • 'git checkout -b branch-name': creates a new branch and switches to it
  • 'git branch' to view Git branches
  • Pull request (PR) is used to merge a comparison branch to the base branch.
